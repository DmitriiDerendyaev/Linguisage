## UserController

`RegistrationController` - это Spring REST-контроллер, отвечающий за обработку операций регистрации пользователей и связанных с ней операций.

## API-эндпоинты

### 1. Получить всех пользователей

- **HTTP-метод:** GET
- **Эндпоинт:** `/registration/all`
- **Описание:** Извлекает список всех зарегистрированных пользователей из базы данных.
- **Ответ:** JSON-массив, содержащий объекты пользователей.

### 2. Получить пользователя по ID

- **HTTP-метод:** GET
- **Эндпоинт:** `/registration/byId/{id}`
- **Описание:** Извлекает информацию о пользователе по его уникальному идентификатору (ID).
- **Параметры:**
  - `{id}` (переменная в пути): ID пользователя для извлечения.
- **Ответ:** JSON-объект, представляющий пользователя, или пустой ответ, если пользователь не найден.

### 3. Получить пользователя по адресу электронной почты

- **HTTP-метод:** GET
- **Эндпоинт:** `/registration/byEmail/{email}`
- **Описание:** Извлекает информацию о пользователе по его адресу электронной почты.
- **Параметры:**
  - `{email}` (переменная в пути): Адрес электронной почты пользователя для извлечения.
- **Ответ:** JSON-объект, представляющий пользователя, или пустой ответ, если пользователь не найден.

### 4. Регистрация пользователя

- **HTTP-метод:** POST
- **Эндпоинт:** `/registration/`
- **Описание:** Регистрирует нового пользователя в системе.
- **Тело запроса:** JSON-объект, представляющий пользователя для регистрации. Объект должен содержать данные пользователя, такие как имя пользователя, адрес электронной почты и пароль.
- **Ответ:** ID только что зарегистрированного пользователя.

### 5. Удалить пользователя по ID

- **HTTP-метод:** DELETE
- **Эндпоинт:** `/registration/delete/{id}`
- **Описание:** Удаляет пользователя по его уникальному идентификатору (ID).
- **Параметры:**
  - `{id}` (переменная в пути): ID пользователя для удаления.
- **Ответ:** Сообщение о том, успешно ли удаление пользователя или не найден пользователь с указанным ID.

## Заметки по использованию

- Это API предназначено для регистрации пользователей и управления ими.
- При регистрации нового пользователя предоставьте данные пользователя в теле запроса в виде JSON-объекта.
- Используйте предоставленные эндпоинты для извлечения информации о пользователе, регистрации новых пользователей и удаления пользователей по мере необходимости.
